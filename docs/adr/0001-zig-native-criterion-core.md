# Zig-native Criterion surface

Sigbench will copy Criterion.rs' product surface and semantics, not its Rust API shape: warmup, sample collection, bootstrap analysis, outlier reporting, baseline comparison, CLI, HTML reports, plots, async benchmarks, and profiler hooks are in scope, while Rust-specific macro ergonomics, crates, and `Drop`-driven timing loops are not. Sigbench also has no runtime dependencies, so plots use an in-tree SVG backend and counters use Zig stdlib, direct syscalls, or OS APIs instead of helper tools like gnuplot. This is hard to reverse once users write benchmarks and baseline files, and the trade-off is deliberate: Zig comptime declarations, preallocated structure-of-arrays buffers, and direct platform integration fit Zig 0.16.0 better than porting Criterion.rs' object-heavy, allocation-friendly internals.
